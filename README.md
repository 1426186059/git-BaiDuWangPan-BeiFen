# GitHub → 百度网盘 自动备份工具

将 GitHub 仓库下载为 ZIP 并自动上传至百度网盘，全程服务端处理，无需中转。

---

## 软件包内容

将压缩包解压到任意目录，得到：

```
BaiduBackup/
├── BaiduBackup.exe          # 主程序（双击运行）
├── web.config               # 端口/服务配置
└── wwwroot/                 # 网页资源（勿删勿改）
    ├── index.html
    └── callback.html
```

---

## 启动

**Windows：** 双击 `BaiduBackup.exe`。

允许防火墙访问提示后，浏览器打开 **http://localhost:5000**。

> 修改端口：用记事本打开 `web.config`，找到 `5000` 改为其他端口，重启程序。  
> 后台运行：将程序放入 Windows 启动文件夹即可开机自启。

**Linux / macOS：**

```bash
chmod +x BaiduBackup
./BaiduBackup
```

> 修改端口：`ASPNETCORE_URLS=http://0.0.0.0:8080 ./BaiduBackup`

## 使用步骤

### 第一步：获取百度网盘 API 凭证

1. 打开 [百度网盘开放平台控制台](https://pan.baidu.com/union/console/home)
2. 创建应用，获取 **AppKey** 和 **SecretKey**
3. 在控制台「安全设置 → 授权回调页」填入 `http://localhost:5000/callback.html`

### 第二步：授权

打开网页 `http://localhost:5000`：

- 填入 AppKey、SecretKey
- 点「授权百度网盘」→ 在弹出的百度页面确认授权 → 自动跳回

### 第三步：选择仓库并备份

- 填写 GitHub 用户名（私有仓库需填 GitHub Token）
- 点「获取仓库列表」
- 勾选要备份的仓库 → 点「逐个备份选中仓库」

备份记录保存在 `backup-records.json`，重复运行默认跳过已备份仓库。

## 常见问题

| 问题 | 解决 |
|------|------|
| 无法访问 GitHub | 页面右上角「网络诊断」检查连通性 |
| 百度授权失败 | 确认回调地址与控制台填写完全一致 |
| Token 过期 | 重新点「授权」即可 |
| 大仓库超时 | 后端有自动重试机制（最多 15 次），耐心等待 |
| 文件已存在跳过 | 网盘已有同名文件时会自动跳过 |
